|    分类    |  范围   |          说明          |       示例码        |

### 2.1 基础信息

### 2.2 用户注册

#### 2.2.1 基本信息

- 接口路径：`/register`
- 请求方式：POST
- 接口描述：用于用户账号注册，校验账号与邮箱唯一性，密码加密存储

#### 2.2.2 请求参数

- 格式：application/json

- 参数说明：

  |  参数名  |  类型  | 必填 |                          说明                          |
  | :------: | :----: | :--: | :----------------------------------------------------: |
  | username | string |  是  |             用户名，唯一标识，长度 3-20 位             |
  | password | string |  是  | 登录密码（8-20 位，包含字母 + 数字，建议前端加密传输） |
  |  email   | string |  是  |            邮箱，唯一标识，符合邮箱格式规范            |
  | nickname | string |  否  |             用户昵称，不传默认与用户名一致             |

  

- 参数样例

  json

  ```json
  {
    "username": "codeuser01",
    "password": "Code123456",
    "email": "codeuser01@example.com",
    "nickname": "刷题小能手"
  }
  ```

  

#### 2.2.3 响应数据

- 格式：application/json

- 参数说明：

  | 参数名 |  类型  |  必填  |              说明              |
  | :----: | :----: | :----: | :----------------------------: |
  |  code  | number |  必须  | 响应码，200 成功，400 代表失败 |
  |  msg   | string | 非必须 |            提示信息            |

  

- 参数示例：

  - 成功

    ```json
    {
      "code": 200,
      "message": "注册成功"
    }
    ```

  - 失败

    ```json
    {
      "code": 400,
      "message": "该用户名已被注册，请更换"
    }
    ```

    

### 2.3 用户登录



#### 2.3.1 基本信息

- 接口路径：`/login`
- 请求方式：POST
- 接口描述：用户名 / 邮箱 + 密码登录方式，签发 JWT Token 实现身份认证



#### 2.3.2 请求参数

- 格式：application/json

- 参数说明：

  |  参数名  |  类型  | 必填 |   说明   |
  | :------: | :----: | :--: | :------: |
  | username | string |  是  |  用户名  |
  | password | string |  是  | 登录密码 |

  

- 参数样例

  ```json
  {
    "username": "codeuser01",
    "password": "Code123456"
  }
  ```

  

#### 2.3.3 响应数据

- 格式：application/json

- 

- 参数说明：

  |    参数名     |  类型  |  必填  |               说明                |
  | :-----------: | :----: | :----: | :-------------------------------: |
  |     code      | number |  必须  |  响应码，200 成功，400 代表失败   |
  |      msg      | string | 非必须 |             提示信息              |
  |     data      | object |  必须  |            返回的数据             |
  |  \|- userId   | number |  必须  |              用户 ID              |
  |    \|-role    | string |  必须  | 角色，user 普通用户 /admin 管理员 |
  |  \|-nickname  | string |  必须  |             用户昵称              |
  |   \|-avatar   | string |  必须  |             头像 URL              |

- 参数示例：

  - 成功

    ```json
    {
      "code": 200,

      "data": {
        "userId": 10001,
        "role": "user",
        "nickname": "刷题小能手",
        "avatar": "https://example.com/avatar/default.jpg",
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "expireTime": "2024-05-01 10:00:00"
      }
    }
    ```

    

  - 失败

    ```json
    {
      "code": 400,
      "message": "账号或密码错误",
      "data": null
    }
    ```

### 3.8 获取用户历史提交记录列表

#### 3.8.1 基本信息

- 接口路径：`/submission/list`
- 请求方式：GET
- 接口描述：获取当前用户的代码提交历史记录列表，支持分页和状态筛选

#### 3.8.2 请求参数

- 格式：Query 参数

- 参数说明：

  表格

  |     参数名      |     类型      | 必填 |                         说明                         |
  | :-------------: | :-----------: | :--: | :--------------------------------------------------: |
  |      page       |    number     |  否  |                     页码，默认 1                     |
  |      size       |    number     |  否  |                  每页条数，默认 10                   |
  |     result      |    number     |  否  | 评测结果：0 - 通过 1- 编译错误 2 - 运行错误 3 - 超时 |
  |     keyword     |    string     |  否  |          题目关键词搜索（题目标题 / 描述）           |
  | problemCategory | Array<string> |  否  |                       题目类别                       |
  |    sortOrder    |    string     |  否  |            按提交时间排序 desc (默认),asc            |
  
- 参数样例:

  plaintext

  ```
  /api/v1/user/submission/list?page=1&size=10&result=2&sortOrder=desc
  ```

  

#### 3.8.3 响应数据

- 格式：application/json
